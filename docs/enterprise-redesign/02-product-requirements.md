# المرحلة الثانية: متطلبات المنتج (PRD)

## Product Requirements Document — Brand Architect AI Pro v2.0

---

## 1. Vision

> **"تمكين كل شركة من أن تمتلك قدرات فريق تسويق كامل باستخدام الذكاء الاصطناعي"**

نهدف إلى بناء أقوى منصة SaaS لإنشاء وإدارة الهوية التجارية والمحتوى التسويقي المدعوم بالذكاء الاصطناعي، تخدم من الشركات الناشئة حتى المؤسسات الكبرى.

---

## 2. Goals

### أهداف الأعمال
- الوصول إلى 100,000 مستخدم مسجل خلال 12 شهراً
- تحقيق $1M ARR خلال 18 شهراً
- معدل احتفاظ بالمستخدمين 80%+ شهرياً
- Net Promoter Score (NPS) 50+

### أهداف المنتج
- زمن إنشاء Brand Kit كامل: أقل من 3 دقائق
- زمن توليد صورة: أقل من 15 ثانية (p95)
- وقت التشغيل: 99.9% uptime
- زمن استجابة API: أقل من 200ms (p95) للـ non-AI requests

### أهداف تقنية
- دعم 10,000 مستخدم متزامن
- معالجة مليون طلب/يوم
- استرجاع البيانات في أقل من 100ms
- قابلية التوسع الأفقي (Horizontal Scaling)


## 5. Functional Requirements

### FR-1: إدارة الهوية
- [x] إنشاء/تعديل/حذف علامات تجارية
- [x] رفع الشعار بصيغ (PNG/JPG/SVG/WEBP) حتى 5MB
- [x] استخراج تلقائي لـ 5 ألوان من الشعار
- [x] Brand Kit: شخصية، تموضع، لهجة، جمهور، أسلوب بصري، لوحة ألوان
- [x] Brand Story وصف ملهم وقصة العلامة
- [x] قوالب Brand Kit قابلة للتخصيص

### FR-2: إدارة الحملات
- [x] إنشاء حملة من brief نصي
- [x] تحديد عدد الأيام (1-30 يوم)
- [x] اختيار المنصات (Instagram/Facebook/Twitter/LinkedIn/TikTok)
- [x] توليد غير متزامن مع تتبع التقدم
- [x] تعديل وحذف وإعادة توليد أي منشور
- [x] تصدير الحملة بصيغ مختلفة
- [x] أرشفة الحملات القديمة

### FR-3: توليد الصور
- [x] دعم أحجام: 1024x1024, 1024x1536, 1536x1024
- [x] دعم نماذج: DALL-E 3, GPT-Image-1, Gemini Imagen
- [x] تحسين البروموبت بثلاثة مستويات
- [x] دعم مراجع متعددة (لوغو + صور)
- [x] سجل تاريخي لكل صورة مولّدة
- [x] استعادة أي صورة سابقة
- [x] توليد جماعي لصور الحملة كاملة

### FR-4: Organizations & Teams
- [x] إنشاء Organization مع domain verification
- [x] دعوة أعضاء عبر البريد الإلكتروني
- [x] أدوار: Owner, Admin, Editor, Viewer
- [x] مساحات عمل (Workspaces) منفصلة
- [x] مشاركة Brand Kits بين الأعضاء
- [x] Audit Log لجميع الأنشطة

### FR-5: الاشتراكات
- [x] خطط: Free, Starter, Professional, Business, Enterprise
- [x] نظام Credits مع حدود شهرية
- [x] تجديد تلقائي عبر Stripe
- [x] Webhooks للأحداث المالية
- [x] فاتورة PDF قابلة للتحميل
- [x] API Keys للوصول المباشر

## 6. Non-Functional Requirements

### الأداء (Performance)
```yaml
API Response Time:
  - P50: < 100ms (non-AI)
  - P95: < 200ms (non-AI)
  - P99: < 500ms (non-AI)
  - AI Text Generation: < 10s
  - AI Image Generation: < 30s

Throughput:
  - 1,000 RPS في الظروف العادية
  - 10,000 RPS في ذروة الاستخدام

Availability:
  - 99.9% uptime (SLA)
  - RTO (Recovery Time Objective): < 15 minutes
  - RPO (Recovery Point Objective): < 1 hour
```

### التوسع (Scalability)
```yaml
Users:
  - 100,000 مستخدم مسجل
  - 10,000 مستخدم متزامن
  - نمو 10x بدون تغيير معماري

Data:
  - 1 مليون Brand
  - 10 مليون Post
  - 100 مليون صورة مولّدة
  - نمو مستمر بدون تدهور في الأداء
```

### الأمن (Security)
```yaml
Authentication:
  - JWT + Refresh Tokens
  - OAuth2 (Google, GitHub)
  - MFA (Multi-Factor Authentication)
  - Session Management

Authorization:
  - RBAC (Role-Based Access Control)
  - Resource-level Permissions
  - API Key Authentication
  - Tenant Isolation

Data Protection:
  - Encryption at Rest (AES-256)
  - Encryption in Transit (TLS 1.3)
  - PII Masking in Logs
  - GDPR Compliance
```

### الجودة (Quality)
```yaml
Code Coverage: > 80%
Bug Fix SLA:
  - Critical: < 4 hours
  - High: < 24 hours
  - Medium: < 1 week
  - Low: < 1 month
Accessibility: WCAG 2.1 AA
Browser Support: Last 2 versions of major browsers
```

## 7. Constraints & Assumptions

### القيود
- يجب الحفاظ على جميع مميزات الإصدار الحالي
- الميزانية تفرض استخدام Cloud providers مقرونة بالتكلفة
- فريق التطوير: 3-5 مطورين
- Timeline: 6 أشهر للـ MVP الجديد

### الافتراضات
- المستخدمون لديهم اتصال إنترنت مستقر
- خدمات AI الخارجية (OpenAI/Gemini) متاحة 99%+
- المستخدمون يقبلون انتظار 15-30 ثانية لتوليد الصور
