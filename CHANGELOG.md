## [2026-05-30] — Admin API Keys, Local Dev, Docs Overhaul, Cleanup

### Backend — Admin AI Provider Key Management
- **New `app/utils/api_key_store.py`** — AI provider key resolver with 60s in-memory TTL cache
  - Reads keys from `app_settings` DB table (key: `"apiKeys"`)
  - Falls back to env vars: `OPENAI_API_KEY` → `GEMINI_API_KEY` → Replit proxy
  - `get_best_provider()` returns `(api_key, base_url, provider_type)` for the first enabled, configured provider
  - `invalidate()` forces DB reload on next call
- **Updated `app/services/ai/client.py`** — now reads from `api_key_store` (not direct env vars)
  - Maintains 60s cached OpenAI client
  - `invalidate_client_cache()` forces new client on next AI call
- **New admin routes in `app/routes/admin.py`** (`/api/admin/api-keys/*`):
  - `GET /admin/api-keys` — list all providers (keys masked to last 4 chars)
  - `POST /admin/api-keys/{provider}` — add/replace key for a provider
  - `DELETE /admin/api-keys/{provider}` — remove a provider's key
  - `POST /admin/api-keys/{provider}/toggle` — enable/disable a provider
  - `POST /admin/api-keys/{provider}/test` — test a key without saving
- **Supported providers:** `openai`, `gemini`, `nano_banana` (custom OpenAI-compatible endpoint)

### Frontend — Admin API Keys Page
- **New `src/pages/AdminApiKeys.tsx`** — full admin UI for managing AI provider keys

  - Configure dialog: add/edit key + base URL (for Nano Banana)

## [2026-05-19] — Security: Rate Limiting + CORS hardening

- **Rate Limiting** added via `slowapi`:
  - `POST /api/auth/login` → 10 req/min per IP
  - `POST /api/auth/register` → 5 req/min per IP
  - `GET /api/auth/me` → 60 req/min per IP
  - Returns `429 Too Many Requests` with `Retry-After` header
- **CORS** restricted from `"*"` to an explicit allowlist:
  - Sources: `REPLIT_DOMAINS` env var (auto-injected by Replit) + localhost variants
  - Override: set `ALLOWED_ORIGINS=https://yourdomain.com` for custom domains
  - Logic lives in `app/config.py → _build_allowed_origins()`
- New file: `app/layers/rate_limit.py` — `limiter` singleton + `rate_limit_exceeded_handler`
